Application Information

This application converts anim.txt, ddt, bti, cub, prt, mtrl, and brg files to user friendly formats for the Age of Mythology game.

Usage

Simply drag, and drop (or use Open With) an anim.txt, prt, and/or mtrl file on the EXE to convert it to an Xml file, or the other way around to convert it accordingly.

The output file will be in the same location as the input file, but with an added extension.

Drag a Brg file on the EXE to extract all of its Mtrl files into a materials folder in the same path as the brg file.

Converting bti/cub to ddt

Texture files can be converted by dropping a bti or cub file on the application executable. If a bti file is dropped, the app expects to find a tga file with the same name. If a cub file is dropped, the app expects to find bti/tga files in the same folder as the cub with the names specified inside of the cub file.

The bti file must specify the number of alpha bits, and the format. For example: alpha=1 fmt=BC3 Avoid using alpha higher than 1 unless you want the game to render semi-transparent models which takes longer. The following formats are supported:

  • BC1, Alpha 0 -- RBG 565 with no alpha. Colors use 4 element palette per 4x4 block.
  • BC1, Alpha 1 -- RBG 565 with alpha 1. Colors use 3 element palette per 4x4 block when the block has alpha, and 4 when it doesn't. Alpha is either on or off.
  • BC2, Alpha 4 -- RGB 565 with alpha 4. Colors use 4 element palette per 4x4 block. Alpha limited to 4 bits and therefore 16 values.
  • BC3, Alpha 8 -- RGB 565 with alpha 8. Colors use 4 element palette per 4x4 block. Alpha uses 8 element palette per 4x4 block. Typically prefer this over BC2.
  • BC3, Alpha 1 -- RGB 565 with alpha 1. Colors use 4 element palette per 4x4 block. Use this when you want higher quality colors than BC1, Alpha 1 at the expense of doubling the file size.

In addition to alpha and format, the following properties can be specified in the bti file:

  • nomip -- Disables generation of mip levels.
  • noalphatest -- Tells the renderer not to make invisible the parts of the texture where the alpha is 0. This is typically used when the alpha represents where to use player color.
